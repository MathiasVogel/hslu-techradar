== Bausteinsicht
=== Level 2: Container-Diagramm

image::c2-diagram.png[C4-Container-Diagramm, scale=false]

[cols="1,3", options="header"]
|===
| Element | Beschreibung
| Angular SPA | Single-Page-Application für Radar-Visualisierung und Administration (Angular, TypeScript, Tailwind, DaisyUI).
| REST API | Backend-Server mit REST-Endpunkten für die Verwaltung der Technologien (Express.js, Node.js).
| Datenbank | PostgreSQL-Datenbank zur Persistierung der Technologiedaten.
| Auth0 | Externer Identity Provider für Authentifizierung via OAuth 2.0 / OIDC.
|===

=== Level 3: Component-Diagramme
==== Client

image::c3-diagram-client.png[C4-Component-Diagramm des Clients, scale=false]

[cols="1,3", options="header"]
|===
| Element | Beschreibung
| NavbarComponent | Navigation zwischen den Seiten und Logout-Funktion.
| HomeComponent | Startseite der Applikation.
| RadarComponent | Visualisierung des Technologie-Radars mit publizierten Technologien.
| RadarAdministrationComponent | Verwaltungsoberfläche für Technologien (Erstellen, Bearbeiten, Publizieren, Löschen).
| TechFormComponent | Formular zum Erfassen, Bearbeiten und Publizieren von Technologien.
| TechDetailComponent | Detailansicht einer einzelnen Technologie.
| TechService | Angular Service für State-Management und API-Kommunikation.
| OpenAPI Client | Typisierter HTTP-Client (`openapi-fetch`) für die Kommunikation mit der REST API.
|===

==== Server

image::c3-diagram-server.png[C4-Component-Diagramm des Servers, width=100%]

[cols="1,3", options="header"]
|===
| Element | Beschreibung
| TechRoutes | Express Router mit REST-Endpunkten (GET, POST, PATCH, DELETE).
| TechController | Verarbeitet eingehende Requests und validiert die Eingaben.
| TechService | Enthält die Business-Logik für Technologie-Operationen.
| Prisma Client | ORM-Schicht für den typsicheren Datenbankzugriff auf PostgreSQL.
| Swagger | Generiert die interaktive API-Dokumentation aus den Route-Definitionen.
|===
