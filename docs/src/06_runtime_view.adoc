== Laufzeitsicht

Das folgende Sequenzdiagramm zeigt die wichtigsten Laufzeitszenarien der Applikation:

image::runtime-diagram.png[Laufzeit-Sequenzdiagramm, scale=false]

[cols="1,3", options="header"]
|===
| Szenario | Beschreibung
| Anmeldung
|Der User öffnet die App, wird zu Auth0 weitergeleitet und erhält nach erfolgreichem Login ein JWT-Token.

| Radar anzeigen
| GET-Request an die API lädt alle Technologien. Der Client filtert und zeigt nur publizierte Einträge als Radar an.

| Technologie erfassen
| POST-Request sendet Formulardaten an die API, die via TechService einen neuen Eintrag in der Datenbank anlegt.

| Technologie publizieren
| PATCH-Request setzt `published: true` inkl. Ring und Begründung. Der TechService ergänzt automatisch das `publishedAt`-Datum.

| Technologie ändern
| PATCH-Request aktualisiert die gewünschten Felder einer bestehenden Technologie.
|===
