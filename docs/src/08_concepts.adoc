== Querschnittliche Konzepte
=== API-Design
Die API folgt REST-Prinzipien mit klaren Ressourcen (z.B. `/technologies`) und HTTP-Methoden (GET, POST, PATCH) für die jeweiligen Operationen.

==== API-Dokumentation
Die API-Dokumentation findet man unter `http://localhost:3000/api-docs` (nach Start des Servers).

=== Testkonzept

Aufgrund der geringen Logikdichte der Einzelkomponenten (CLient und Server) und zur Vermeidung von aufwendigem Mocking wurde eine pragmatische, E2E-zentrierte Teststrategie gewählt.

- *Schwerpunkt E2E (Cypress):* Die 7 User Stories werden direkt über den Browser validiert. Dies stellt das korrekte Zusammenspiel von Frontend, API und Datenbank ohne Mocks sicher und bietet das beste Kosten-Nutzen-Verhältnis im gegebenen Zeitrahmen.

- *Unit-Tests (Vitest, Supertest):* Ergänzende Absicherung isolierter Logik. Hierbei werden Abhängigkeiten konsequent gemockt, um die Testausführung schnell und unabhängig von der Infrastruktur zu halten.
