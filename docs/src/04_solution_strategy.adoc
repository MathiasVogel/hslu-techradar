[[section-solution-strategy]]
== Lösungsstrategie

[cols="2,3,3", options="header"]
|===
| Ziel / Anforderung | Lösungsansatz | Umsetzung

| **Client-Server-Architektur**
| Trennung von Frontend und Backend als eigenständige Applikationen
| Angular (Client) kommuniziert via REST-API mit Express/Node.js. Typsicherheit durch OpenAPI-Generierung.

| **Datenpersistenz**
| Relationale Datenbank mit typsicherem ORM
| PostgreSQL als Datenbank, Prisma ORM für Schema-Management, Migrationen und typsicheren Datenbankzugriff.

| **Authentifizierung & Autorisierung**
| Delegierte Authentifizierung über einen externen Identity Provider
| Auth0 (OAuth 2.0 / OIDC) übernimmt Login und Rollenverwaltung (CTO/Mitarbeiter).

| **Responsive Darstellung**
| Mobile-/Tablet-optimierter Viewer
| Tailwind CSS mit responsiven Breakpoints für Desktop und Mobile.

| **Testabdeckung**
| Qualitätssicherung durch automatisierte Tests auf mehreren Ebenen
| E2E-Tests mit Cypress; Unit- und Integrationstests mit Vitest und Supertest.

| **Containerisierung**
| Reproduzierbare Entwicklungs- und Deployment-Umgebung
| Docker Compose orchestriert Client, Server und Datenbank.

| **API-Dokumentation**
| Selbstdokumentierende API
| Swagger UI generiert interaktive Dokumentation direkt aus dem Backend-Code.
|===
